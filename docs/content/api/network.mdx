# Network

网络工具函数，用于检测网络接口、获取局域网 IP 地址以及检查端口可用性。

## 依赖说明

`ensure_port_available` 为标准库实现，无需额外依赖。

:::warning psutil
`get_private_networks` 依赖 `psutil`。使用前请在插件的 `pyproject.toml` 中声明：

```toml
dependencies = [
    "psutil>=5.9.0",
]
```

缺失时 bykpy 会抛出清晰的 `ImportError` 并提示安装命令。
:::

## get_private_networks

获取所有局域网 IP 地址信息，按优先级排序。

<Tabs>
<Tab label="函数签名">
```py
def get_private_networks() -> List[Dict]:
    """获取所有局域网 IP 地址信息。
    
    Returns:
        包含网络接口信息的字典列表，每个字典包含：
        - iface: 网络接口名称
        - ips: IP 地址列表
        - type: 接口类型
        - virtual: 是否为虚拟接口
        - priority: 优先级（数字越小优先级越高）
    """
```
</Tab>
<Tab label="返回值示例">
```py
[
    {
        "iface": "eth0",           # 网络接口名称
        "ips": ["192.168.1.100"],  # IP 地址列表
        "type": "ethernet",        # 接口类型
        "virtual": False,          # 是否虚拟接口
        "priority": 10             # 优先级
    },
    {
        "iface": "wlan0",
        "ips": ["192.168.1.101"],
        "type": "wifi",
        "virtual": False,
        "priority": 10
    },
    {
        "iface": "docker0",
        "ips": ["172.17.0.1"],
        "type": "container",
        "virtual": True,
        "priority": 40
    }
]
```
</Tab>
</Tabs>

### 接口类型说明

系统支持以下接口类型分类：

| 类型 | 关键词 | 虚拟接口 | 优先级 | 说明 |
|------|--------|---------|--------|------|
| ethernet | ethernet, 以太网 | False | 10 | 有线以太网 |
| wifi | wlan, wi-fi, 无线 | False | 10 | 无线网络 |
| vmware | vmware, vmnet | True | 30 | VMware 虚拟网卡 |
| virtualbox | vbox, virtualbox | True | 30 | VirtualBox 虚拟网卡 |
| container | docker, wsl | True | 40 | 容器网络 |
| bluetooth | bluetooth | True | 60 | 蓝牙网络 |
| loopback | loopback | True | 100 | 回环接口 |
| unknown | - | False | 50 | 未知类型 |

### 使用示例

<Tabs>
<Tab label="基础使用">

```py
from bykpy.api import get_private_networks

# 获取所有局域网接口
networks = get_private_networks()

for net in networks:
    print(f"接口: {net['iface']}")
    print(f"类型: {net['type']}")
    print(f"IP: {', '.join(net['ips'])}")
    print(f"虚拟: {'是' if net['virtual'] else '否'}")
    print(f"优先级: {net['priority']}")
    print()
```

</Tab>
<Tab label="获取首选 IP">

```py
from bykpy.api import get_private_networks

def get_preferred_ip() -> str:
    """获取首选的局域网 IP。"""
    networks = get_private_networks()
    if networks:
        # 返回优先级最高的接口的第一个 IP
        return networks[0]['ips'][0]
    return "127.0.0.1"

# 使用
host = get_preferred_ip()
print(f"服务器将绑定到: {host}")
```

</Tab>
<Tab label="过滤虚拟接口">

```py
from bykpy.api import get_private_networks

def get_physical_networks():
    """获取物理网络接口。"""
    networks = get_private_networks()
    return [net for net in networks if not net['virtual']]

# 使用
physical = get_physical_networks()
for net in physical:
    print(f"{net['iface']}: {net['ips']}")
```

## ensure_port_available

检查指定端口是否可用。

<Tabs>
<Tab label="函数签名">
```py
def ensure_port_available(port: int, host: str = "0.0.0.0") -> None:
    """
    Args:
        port: 要检查的端口号
        host: 绑定的主机地址，默认为 "0.0.0.0"
        
    Raises:
        OSError: 如果端口已被占用或无法绑定
    """
```
</Tab>
<Tab label="使用示例">
```py
import click
from bykpy.api import ensure_port_available, pass_command_context

@click.command()
@click.option("--port", default=8080, help="服务器端口")
@pass_command_context
def server(ctx, port):
    """启动 HTTP 服务器"""
    try:
        # 检查端口是否可用
        ensure_port_available(port)
        ctx.logger.info(f"端口 {port} 可用，正在启动服务器...")
        
        # 启动服务器逻辑
        # start_server(port)
        
    except OSError:
        ctx.logger.error(f"端口 {port} 已被占用，请选择其他端口")
        raise click.Abort()
```
