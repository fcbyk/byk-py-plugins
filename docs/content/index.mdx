import click

<Tab label="main.py">

```py
from bykpy import PluginProtocol

class Plugin(PluginProtocol):
    """一个插件可以声明多个子命令。"""

    commands = {
        "hello": "第一个子命令的描述",
        "world": "第二个子命令的描述",
    }

    def register(self, cli):
        from .cli import hello, world  # 延迟 import，按需加载
        cli.add_command(hello)
        cli.add_command(world)
```

## 插件注册机制

插件通过 Python entry points 注册，entry point 指向一个继承 `PluginProtocol` 的类

### PluginProtocol

插件类仅需两个成员

| 成员 | 类型 | 说明 |
|------|------|------|
| `commands` | `dict[str, str]` | 子命令名 → 描述，用于仪表盘展示 |
| `register(self, cli)` | 实例方法 | 接收 `click.Group`，在此调用 `cli.add_command()` 注册子命令 |

### 按需加载

byk 启动时仅读取 `commands` dict 构建缓存（O(1) 次 IO），不实例化插件。用户执行对应子命令时才触发 `import → 实例化 → register`。

:::tip 延迟导入
建议在 `register` 方法内部 `from .cli import ...` 延迟导入，避免启动时加载不必要的依赖。
:::

### 同名命令冲突

多个插件声明同名命令时，后加载的覆盖前者，并输出 warning 日志。

## 依赖模型

bykpy 遵循「谁使用谁负责依赖」原则：

- **运行时仅依赖 `click`**，其他 API 所需的第三方库由调用方（插件）自行声明
- 如果插件使用了某个 API 但缺少对应依赖，bykpy 会抛出清晰的 `ImportError` 并提示 `pip install` 命令——**不会静默失败**

| 你的插件用了... | 需要在 pyproject.toml 加 |
|---|---|
| `create_spa` / `R` / `get_client_ip` | `flask>=2.0` |
| `copy_to_clipboard` | `pyperclip>=1.9.0` |
| `get_private_networks`（生产环境） | `psutil>=5.9.0` |
| 仅框架 API | 无需额外依赖 |

:::tip 依赖守卫示例
调用 `copy_to_clipboard` 但未安装 pyperclip 时：

```
ImportError: pyperclip is required for clipboard operations.
Install it with: pip install pyperclip
```
:::
